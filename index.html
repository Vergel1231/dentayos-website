The Problem

Most small dental and aesthetic clinics in Metro Manila still rely on paper records, manual scheduling, and group chats to run day-to-day operations. This leads to missed appointments, lost patient history, and staff spending hours on admin work instead of patient care.

Existing solutions are either too expensive, built for hospitals, or imported foreign software that doesn't fit the Filipino clinic context.

The Solution

DentAyos is a dual-portal web application — one system with two sides. The clinic portal gives staff a dashboard to manage records, appointments, and reminders. The patient portal lets patients self-register, book appointments, and view their own treatment history. Both portals are accessible from any device with a browser, with no app installation required.

Key Design Decisions

Dual portal architecture — Clinic staff and patients have separate logins, separate dashboards, and see only the data relevant to them.

localStorage sync — Patient-booked appointments sync to the clinic portal in real time using browser localStorage, enabling a full booking workflow without a backend. Note: this means data is browser-specific — different browsers or incognito mode will show an empty state. This is an intentional prototype trade-off, not a bug.

Pilot-first pricing — The product was priced to land the first paying clinic, not to maximize margin. The pilot structure funds the next development phase.

Filipino-first UX — Mixed Tagalog-English copy, peso pricing, and local clinic workflows were prioritized over generic international patterns.

What's Next

The current prototype demonstrates the full product workflow. The next phase connects it to a real backend with Firebase authentication, persistent database storage, and SMS reminders via Twilio.
